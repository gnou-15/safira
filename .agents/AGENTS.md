# Workspace Behavior Rules

Please adhere strictly to the following developer-specified rules:

- **Git Control**: Do not perform any Git operations (`git add`, `git commit`, `git push`, etc.). The user will handle all version control staging, committing, and pushing manually.
- **Extreme Modularity**: Every separate function that is considered relevant or reusable should be placed in its own individual file to ensure files remain short, clean, and highly readable.
- **Skip Automated Testing**: Do not propose or run any automated test commands (like Jest, Vitest, Pytest, etc.).
